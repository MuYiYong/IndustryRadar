### Is GraphRAG Needed? From Basic RAG to Graph-/Agentic Solutions with Context Optimization

- 链接：[arXiv:2606.25656](https://arxiv.org/abs/2606.25656)
- 核心问题：比较普通 RAG、GraphRAG、Modular RAG 与 Agentic RAG，并研究上下文优化对成本和效果的影响。
- 关键启示：高级检索并不总能带来等比例的生成质量提升；存在明显的“检索—生成落差”。GraphRAG 产品需要同时关注回答质量、证据解释、上下文成本和生产治理，而不能只比较召回指标。

### Agentic GraphRAG: Navigating Unstructured Financial Data with Collaborative AI

- 链接：[arXiv:2605.18770](https://arxiv.org/abs/2605.18770)
- 核心问题：针对商业登记和金融非结构化数据，构建协作式 Agentic GraphRAG 框架。
- 技术路线：从结构化字段构建强节点，从公告等文本抽取弱节点，进行身份解析和去重，再由多个 Agent 完成意图路由、图工具访问、受限反思和状态感知回答。
- 产业启示：适用于工商、司法公告、监管处罚、财报、舆情和供应链合同等金融数据，可将实体解析、图关系、Agent 协作和证据追踪组合为生产流程。

### Agentic Neural Graph Database

- 核心方向：探索图数据库自主构造查询、神经化执行和持续学习能力。
- 近期可落地判断：生产系统仍应以确定性图查询和图算法负责事实计算，由 LLM 负责意图理解和计划生成，本体与策略负责约束，Dry Run 负责执行前验证。

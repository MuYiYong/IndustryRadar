# Ontology

## 2026-07-20｜W29 知识追加

### 从描述模型到规范模型

Agent 时代的企业本体不只描述客户、账户、合同、产品等业务对象，还需要定义主体能够执行哪些动作、动作需要满足哪些前置条件、哪些数据和工具受到何种政策约束，以及哪些结果必须保留证据。

### Ontology-to-tools

值得关注的路线是把本体规范编译为 Agent 必须调用的工具接口。相比“模型自由生成、事后校验”，这种模式把合法对象、关系、参数和动作边界直接固化到执行接口中，使部分不合规动作无法提交。

### 本体治理闭环

1. 本体定义业务对象、关系、动作和规则；
2. 知识图谱实例化现实主体与状态；
3. 策略引擎根据身份、关系、时间和风险作出 permit、review 或 deny 判断；
4. Agent Dry Run 在隔离环境中模拟动作影响；
5. 运行时记录数据、工具、规则、结果和业务影响；
6. 根据异常、审计和业务变化持续演化本体。

### 核心定位

> 本体是企业 Agent 理解业务世界、判断约束并选择动作的机器可执行语义模型。
